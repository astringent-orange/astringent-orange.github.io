7_小型仿真强化学习实验

Mian 发布于 2026-01-28

  • 📒 无人机实验
  • 🏷️ simulation
  • 🏷️ RL

尝试完成一个微型实验，在Ubuntu 20.04 + ROS noetic的环境中，使用PX4 firmware + mavros + gazebo，GPU nvidia rtx 2060 6G + CPU intel i7-10875H实现一个RL的过程。输入为64×64深度图像，输出为水平速度（高度固定）+ 旋转速度。模型选择基本的CNN处理图像后获得特征向量，与无人机状态向量进行拼接，经过MLP后输出。

环境搭建

构建工作空间，并安装PX4、配置gazebo模型、运行mavros通信。

source覆盖问题

以前下载过PX4 firmware且配置过bashrc文件，这次尝试运行时发现，如果source ./devel/setup.bash之后则会找不到px4，再去source ~/.bashrc则会找不到功能包。
原因是两次source的内容互相覆盖了。解决方法一是修改./devel/setup.bash的内容，在其中加入px4的路径（不推荐修改bashrc文件）；解决方法二，单独为该项目安装一个px4固件。–>相当于增加了一个controller文件夹

补充内容

事后发现这样每次都要下载一个PX4源码，虽然对于打包程序来说很好，但是对于自己本地使用还是不方便。于是提出来了方法三。
发现之前的问题是source bashrc和/devel/setup会分别覆盖对方的结果，更具体而言，是source /devel/setup和PX4/Tools/setup_gazebo.bash时，因为两者不知道对方的层级，所以会清空对方的$ROS_PACKAGE_PATH，然后添加自己的导致的路径导致的。而PX4的路径其实并不是在bash文件中添加的，而是export添加的，所以可以写一个如下的启动脚本set_env.sh，每次source该脚本即可。注意一定是source，而不是sh执行。因为source的作用是将脚本内容复制到当前终端中执行，而sh或者bash是另外开一个终端来执行。

分析该脚本，注意到是最后才去exportPX4的位置，这样就可以避免被刷掉。
